# 项目定位

**项目名：PyPuzzle / Python代码拼图**

## 一句话介绍

一个专门为 Python 初学者设计的代码拼图练习工具。
 通过拖拽或点击排序，把打乱的 Python 基础代码还原成正确顺序，帮助新手理解代码结构和执行逻辑。

------

# 一、MVP目标

第一版只验证三件事：

1. 新手是否愿意通过“拼代码顺序”来练习
2. 基础代码拼图是否能帮助他们更好理解代码结构
3. 游戏化练习是否能提升学习兴趣

所以第一版只做：

**基础代码顺序还原练习**

不做复杂判题，不做真正执行代码，不做高级语法分析。

------

# 二、目标用户

## 核心用户

- Python 初学者
- 会一点基础语法但代码顺序容易乱的人
- 培训班学员
- 刚学完 print、input、if、for 的小白

## 核心痛点

- 看教程时懂，自己写时顺序乱
- 不知道哪一行该放前面
- 不理解代码执行的先后关系
- 循环、判断、输入输出混在一起时容易懵
- 学习太枯燥，缺少互动感

------

# 三、产品核心思路

这个项目的本质不是“拖拽游戏”，而是：

**通过拼接代码，让用户形成对代码结构的直觉。**

也就是训练这些能力：

- 哪一行应该先执行
- 输入、处理、输出的顺序
- 条件判断的结构
- 循环中的代码块关系
- 缩进代码的从属关系

------

# 四、MVP玩法设计

我建议第一版做 **一个主玩法** 就够：

## 主玩法：代码顺序拼图

给用户一段被打乱顺序的基础 Python 代码，让他恢复正确顺序。

例如：

原始正确代码：

```
name = input("请输入姓名")
print("你好")
print(name)
```

打乱后显示成：

```
print(name)
name = input("请输入姓名")
print("你好")
```

用户需要把它调整回正确顺序。

# 五、MVP功能范围

## 第一版必须有的功能

### 1. 首页

内容包括：

- 项目标题
- 简介
- 开始练习按钮
- 继续练习按钮
- 当前进度摘要
- 选择题目类型

### 首页文案示例

标题：
 **Python代码拼图**

副标题：
 **把打乱的代码拼回正确顺序，边玩边理解 Python。**

按钮：

- 开始闯关
- 继续上次练习

进度摘要：

- 已完成 8 题
- 当前关卡：循环练习
- 最高连胜：5 题

------

### 2. 拼图练习页

这是核心页面。

页面内容：

- 当前关卡名
- 题号
- 题目说明
- 打乱后的代码块列表
- 操作按钮
- 提交按钮
- 重置按钮

例如：

题目说明：
 **请将下面代码恢复成正确顺序，使程序先获取姓名，再输出问候。**

代码块：

- `print(name)`
- `name = input("请输入姓名")`
- `print("你好")`

按钮：

- 上移
- 下移
   或者
- 拖拽排序

------

### 3. 判题结果

用户点击“提交”后，系统判断顺序是否正确。

### 如果正确

显示：

- 拼图成功
- 正确代码展示
- 代码作用解释
- 下一题按钮

### 如果错误

显示：

- 顺序不正确
- 提示哪几行位置有问题
- 可重新调整
- 查看答案按钮（第一版可以后加）

------

### 4. 基础解释区

每道题在提交后，都显示一个简短解释：

例如：

- 先让用户输入姓名
- 再输出固定问候语
- 最后输出用户输入的姓名

这个解释能加强理解，不然拼图容易变成机械排序。

------

### 5. 关卡分类

第一版建议分 4 类基础题：

#### A. 输入输出类

- print
- input
- 变量保存
- 输出结果

#### B. 变量与计算类

- 变量赋值
- 简单加法
- 结果输出

#### C. 判断类

- if / else
- 比较结果
- 条件输出

#### D. 循环类

- for
- range
- print

每类 5~10 题。

------

### 6. 学习进度保存

使用 `localStorage` 保存：

- 当前题目进度
- 已完成题目
- 当前分类
- 连续答对数
- 最高分 / 用时
- 错题记录（可选）

------

### 7. 手机端适配

必须支持手机浏览器。

手机端建议：

- 使用单列布局
- 每个代码块做成大卡片
- 不强依赖拖拽，最好支持“上移/下移”按钮
- 按钮足够大

这里很关键：
 **纯拖拽在手机端体验不一定稳定，MVP 阶段建议优先用“上移 / 下移”排序。**

# 九、题目数据设计

题目建议写在 `data.js` 中。

## 示例结构

```
const puzzles = [
  {
    id: 1,
    category: "输入输出",
    title: "输入姓名并输出",
    description: "请将代码恢复成正确顺序，使程序先获取姓名，再输出问候。",
    correctOrder: [
      'name = input("请输入姓名")',
      'print("你好")',
      'print(name)'
    ],
    explanation: "这段代码先获取用户输入的姓名，然后输出问候语，最后输出姓名。"
  },
  {
    id: 2,
    category: "循环",
    title: "循环输出数字",
    description: "请将代码恢复成正确顺序，使程序循环输出 0 到 2。",
    correctOrder: [
      'for i in range(3):',
      '    print(i)'
    ],
    explanation: "这段代码会循环 3 次，并依次输出 0、1、2。"
  }
];
```

运行时再把 `correctOrder` 打乱后展示。

# 十、交互方式设计

第一版建议你用 **点击排序**，不要先做拖拽。

## 推荐方式 1：上移 / 下移

每个代码块旁边有：

- ↑ 上移
- ↓ 下移

优点：

- 好开发
- 手机端稳定
- 新手容易理解

## 推荐方式 2：点击交换

用户先选中一行，再选另一行交换位置。

这个也可以，但不如上移 / 下移直观。

## 不建议第一版就用纯拖拽

因为：

- 手机端兼容性和体验容易翻车
- 开发成本更高
- 新手未必更好用

# 十一、核心用户流程

## 第一次使用

打开首页
 → 点击开始练习
 → 进入第一题
 → 调整代码顺序
 → 点击提交
 → 查看结果和解释
 → 进入下一题

## 再次使用

打开网页
 → 从 localStorage 读取进度
 → 点击继续练习
 → 回到上次题目

## 学习强化

做完题后
 → 看解释
 → 记住这段代码的执行顺序
 → 继续下一题

# 十五、手机端需求怎么满足

既然你强调手机端，这个项目在交互上必须照顾手机。

## 手机端建议

### 1. 卡片式代码块

每一行代码做成独立卡片，圆角、边框清楚。

### 2. 操作按钮大

每张卡片底部有：

- ↑ 上移
- ↓ 下移

### 3. 单列布局

全部纵向排列。

### 4. 提交按钮固定明显

放在底部或页面明显位置。

### 5. 代码字体清晰

使用等宽字体，方便认代码结构。

# 十六、UI方向建议

这个项目适合的风格：

- 学习工具风 + 轻游戏感
- 清爽
- 卡片化
- 层次明确
- 代码区和说明区明显区分

## 色彩建议

- 主色：蓝色
- 成功：绿色
- 错误：红色

## 视觉重点

- 代码块本身要突出
- 行与行之间区分清晰
- 成功/失败反馈明显
- 说明区尽量简洁

# 十七、MVP验收标准

只要满足这些，就算 MVP 成立：

1. 双击 `index.html` 能直接打开

# 二十一、一句话总结

这个项目的 MVP 本质上是：

**一个无需安装、双击即用、支持手机端的 Python基础代码拼图练习工具。**
